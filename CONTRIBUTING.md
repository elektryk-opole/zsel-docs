# Contributing to zsel-docs

Thank you for helping improve our documentation!

## Getting Started

1. **Fork** the repository
2. **Clone** your fork: `git clone https://github.com/YOUR-USERNAME/zsel-docs.git`
3. **Create a branch**: `git checkout -b docs/topic-name`
4. **Edit** documentation files
5. **Submit a pull request**

## Documentation Guidelines

### Structure
- Use clear headings (H1, H2, H3)
- Include table of contents for long docs
- Use consistent formatting
- Link to related documents

### Content
- Write for multiple audiences (operators, developers, auditors)
- Include examples and code snippets
- Add diagrams for complex concepts
- Update docs when code changes

### Formatting
- Use Markdown (.md format)
- Include metadata in YAML front matter
- Use code blocks with syntax highlighting
- Add navigation breadcrumbs

### Examples
- Include complete, runnable examples
- Add command-line examples
- Show expected output
- Document common errors

## Pull Request Process

1. **Preview locally**: Build docs and verify formatting
2. **Check links**: Ensure all links are valid
3. **Add metadata**: Include author, date, version
4. **Request review**: Assign documentation maintainers
5. **Address feedback**: Implement suggestions
6. **Merge**: Maintainers will merge after approval

## Documentation Standards

### File Organization
```
topic/
├── OVERVIEW.md       Navigation and overview
├── SETUP.md          Installation and configuration
├── USAGE.md          How to use the feature
├── TROUBLESHOOTING.md Common issues and solutions
└── REFERENCE.md      Complete API/config reference
```

### Code Examples
- Must be copy-paste ready
- Include all imports and dependencies
- Show both success and error cases
- Add explanatory comments

### Diagrams
- Use Mermaid for flowcharts
- Include descriptions
- Keep diagrams up-to-date
- Vector format preferred

## Reporting Issues

Found incorrect information? Open an issue with:
- Location in documentation
- What's incorrect
- What should it be
- Any relevant links or references

## Adding New Documentation

### New Topic Checklist
- [ ] Create topic directory
- [ ] Write OVERVIEW.md with navigation
- [ ] Include setup/installation instructions
- [ ] Add usage examples
- [ ] Document common issues
- [ ] Link to related topics
- [ ] Add to main documentation index

### Translation
Help translate documentation:
1. Create `docs/[language-code]/`
2. Copy English structure
3. Translate content
4. Maintain formatting and links

## Writing Tips

- Be concise but complete
- Use active voice
- Write for the reader's perspective
- Break into small sections
- Use examples liberally
- Update regularly
